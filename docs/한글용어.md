# 한글 용어 사용 규칙 — englisheducation

> **`_study.html` 개념 정리, OX 퀴즈 해설(`why` 필드), 챕터 도입부 등 한글이 등장하는 모든 본문에 적용되는 용어 정책.**
> 핵심: 한글 설명은 유지하되, 그 안의 학술 키워드/개념 용어는 영어 원어로 통일한다.
> 임용 관련 모든 레포(testmaster, englisheducation, phonetics-phonology, transformational-grammar, linguistics)에 공통 적용되는 정책. **원본 정책: 전역 `~/.claude/CLAUDE.md`** + **testmaster `docs/한글용어.md`**.

---

## 원칙

1. **한글 설명 자체는 유지** — 수험생을 위한 한글 풀이/요약/맥락 설명은 그대로 둔다. 한글 블록을 통째로 삭제하지 않는다.
2. **키워드 개념 용어는 영어 원어로** — 학술 용어를 한글 번역 조어로 옮기면 어색하므로 영어 원어로 표기한다.
3. **refs(합격자노트·강사 교재 MD·기출 원문) 표현 우선** — refs에 한글 표현이 있으면 그것을 쓴다. 없으면 영어 원어.
4. **혼용 금지** — 같은 개념을 한 페이지 안에서 한글/영어 혼용 금지. 한쪽으로 통일.

## 사용 OK 한글 표현 (자연스러움 / refs 또는 한국 영어교육 학계 표준)

| 한글 | 영어 |
|------|------|
| 화용 능력 | pragmatic competence |
| 발화행위 | speech act |
| 공손 전략 | politeness strategy |
| 비계 | scaffolding |
| 수준 혼합 학급 | mixed-ability classroom |
| 메타인지 | metacognition |
| 메타인지 전략 | metacognitive strategy |
| 학습 전략 | learning strategy |
| 의사소통 능력 | communicative competence |
| 형태 초점 | focus on form |
| 의미 협상 | meaning negotiation |
| 입력·출력·상호작용 | input · output · interaction |
| 임계기 가설 | Critical Period Hypothesis |
| 작업 기억 | working memory |
| 듣기·말하기·읽기·쓰기 | listening · speaking · reading · writing |

### 자연스러운 한글 일반 설명

`구체적 결과물`, `의미 있는 정보 교환`, `숙달도가 낮은 학습자`, `상위 인지 과제`, `정보 구조`, `구조적 복잡성`, `학습자 요구`, `과제 수행`, `상호작용 활동` 등 — 학술 키워드가 아닌 일반 한글 설명은 그대로 사용.

## 사용 금지 한글 번역 조어 → 영어 원어로 교체

> refs에 등장하지 않거나 한국어로 번역하면 어색해지는 학술 용어. 본문에서 단독 사용 시 영어 원어로 통일.

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 의사소통 드릴 / 의사소통 중심 드릴 | communicative drill |
| 정보 간격 / 정보 격차 | information gap |
| 구인 타당도 | construct validity |
| 구인 일치 | construct alignment |
| 차별화 교수 | differentiated instruction |
| 블룸 분류법 | Bloom's taxonomy |
| 대인 의사소통 활동 | interpersonal communication activities |
| 의사소통 수단으로서 언어 사용 | language as communication |
| 기계적 드릴 | mechanical drill |
| 인지 부하 | cognitive load |
| 유도된 출력 | pushed output |
| 알아차림 가설 | Noticing Hypothesis |
| 진정성 있는 자료 | authentic materials |
| 형태 초점 교수 | focus-on-form instruction |
| 후방 설계 | backward design |
| 교사 발화 시간 | teacher talk time |

> 위 표는 대표 사례. 일반 원칙: **refs에 한글 표현이 없으면 → 영어 원어 그대로.**

## 적용 패턴 예시

### Before (어색한 번역 조어 단독 사용)

```html
<p>차별화 교수는 수준 혼합 학급에서 학습자 요구에 대응하는 전략이다.</p>
```

### After (한글 설명 유지 + 학술 키워드 영어로)

```html
<p>differentiated instruction은 수준 혼합 학급에서 학습자 요구에 대응하는 전략이다.</p>
```

## 위반 즉시 수정 트리거

- `_study.html` 본문 또는 OX 퀴즈 `why`에 위 표의 ❌ 항목이 단독 한글로 등장 → 즉시 영어 원어로 교체.
- 영어 키워드 + 한글 글로스(`scaffolding(비계)` 등) 형태에서 한글 글로스가 어색하거나 영어와 동일 의미면 → 한글 글로스 제거.
- refs 어디에도 등장하지 않는 새 한글 학술 번역어를 도입 → 즉시 영어 원어로 교체.
